# 技术栈

## 核心技术

### LangGraph - 状态图驱动引擎
- **用途**: Agent编排和状态管理
- **优势**: 声明式状态机，易于理解和调试
- **特性**: 支持循环、条件分支、并行执行

### LangChain - LLM集成框架
- **用途**: 统一LLM接口和工具管理
- **优势**: 丰富的生态和工具支持
- **特性**: 工具绑定，提示工程，链式调用

### FastAPI - 高性能Web框架
- **用途**: REST API服务
- **优势**: 异步支持，自动文档，类型安全
- **特性**: Pydantic集成，依赖注入，中间件

### Pydantic - 数据验证
- **用途**: 请求/响应数据验证和序列化
- **优势**: 类型安全，自动验证，JSON Schema生成
- **特性**: V2版本性能优化，支持自定义验证器

### SQLite - 状态持久化
- **用途**: 会话状态和上下文存储
- **优势**: 无需额外依赖，文件数据库，ACID事务
- **特性**: 内存模式支持，生产环境可替换为其他数据库

## 支持的LLM提供商

### OpenAI
- **模型**: GPT-4, GPT-3.5-turbo
- **优势**: 业界领先，丰富的训练数据
- **集成**: 原生LangChain支持

### Anthropic
- **模型**: Claude-3系列
- **优势**: 安全性和推理能力
- **集成**: 官方LangChain集成

### SiliconFlow
- **模型**: DeepSeek, Qwen等开源模型
- **优势**: 成本效益高，OpenAI兼容API
- **集成**: 自定义提供商实现
