# 功能特性

## 当前核心功能

- ✅ **基于LangGraph的状态管理**
  - 使用状态图驱动Agent对话流程
  - 支持复杂的多步骤对话逻辑
  - 自动状态转换和错误处理

- ✅ **基础对话能力**
  - 支持OpenAI GPT-4/3.5
  - 支持Anthropic Claude
  - 支持SiliconFlow (兼容OpenAI API)
  - 流式对话和上下文管理

- ✅ **LangChain工具集成**
  - 内置计算器工具
  - 支持自定义工具扩展
  - 自动工具调用和结果处理

- ✅ **MCP (Model Context Protocol) 集成**
  - 支持 stdio 传输的 MCP 服务器
  - 应用启动时预热 MCP 服务连接
  - 自动工具发现和注册
  - 兼容现有 MCP 生态 (filesystem, weather, git 等)
  - 异步生命周期管理和错误处理
  - MCP 服务异常时支持降级运行，不阻塞 Agent 核心启动

- ✅ **Skill 系统（渐进式加载）**
  - 支持 `metadata / full / resources` 三层加载策略
  - 可按场景控制上下文注入粒度与 Token 成本
  - 基于 `SKILL.md` 提供任务级指导与可扩展能力封装

- ✅ **HTTP工具集成**
  - 支持外部API调用
  - 灵活的参数映射
  - 错误处理和重试机制

- ✅ **会话管理和上下文持久化**
  - 基于SQLite的状态持久化
  - 会话隔离和上下文保持
  - 内存存储，生产环境可扩展

- ✅ **自动工具调用和结果处理**
  - LLM自动识别工具调用需求
  - 并行工具执行支持
  - 结果格式化和错误处理

- ✅ **RESTful API接口**
  - FastAPI构建的高性能接口
  - 自动API文档生成
  - 支持异步请求处理

- ✅ **Python SDK**
  - 简洁的Python客户端
  - 异步接口支持
  - 易于集成到现有项目

- ✅ **结构化日志和监控**
  - 基于structlog的结构化日志
  - 请求追踪和性能监控
  - JSON格式日志输出

- ✅ **生产部署能力**
  - 支持 Gateway + Agent 分层部署
  - 支持基于 Redis 的服务发现与多实例负载均衡
  - 支持熔断、健康检查与运行时可观测性

## 当前版本暂不包含的功能

- 🚫 **多语言SDK**
  - 目前仅支持Python
  - 计划支持JavaScript/TypeScript

- 🚫 **多模态输入输出**
  - 当前以文本对话为主
  - 暂不包含音频/视频等多模态链路

- 🚫 **高级安全特性**
  - 自动化 API 密钥轮换
  - 细粒度 RBAC 权限模型
  - 企业级合规审计体系

- 🚫 **企业级多租户能力**
  - 组织/租户级资源隔离
  - 配额计费与租户治理

- 🚫 **复杂监控告警**
  - 标准化告警编排与通知路由
  - 完整仪表板与容量规划体系

## 架构优势

- **双服务解耦**: `zero-gateway` 负责接入与流量治理，`zero-agent` 负责推理编排与工具执行
- **编排驱动执行**: 基于 LangGraph 的状态流工作流，支持多步骤任务与工具协同
- **MCP 降级容错**: 外部 MCP 服务异常时可降级运行，不阻塞核心会话服务启动
- **Skill 渐进式注入**: 支持 `metadata/full/resources` 分级加载，平衡效果与 Token 成本
- **可扩展工具生态**: 统一接入内置工具、HTTP 工具与 MCP 工具，扩展边界清晰
- **可运维部署**: 支持 Redis 服务发现、负载均衡、熔断与结构化日志
